# Backend レビュー評価ロジック変更 報告書

## 変更日
2026-03-16

## 変更概要
レビュー評価を5段階(1-5)から3段階(3,4,5)に変更。
ネガティブ評価を排除し、平均値が荒れにくい構造にした。

## 変更ファイル

### reviews/models.py
- `rating_overall` に `RATING_CHOICES` を追加: `(3, '観た'), (4, '良かった'), (5, '最高')`
- バリデータを `MinValueValidator(3)` に変更

### reviews/serializers.py
- `validate_rating_overall` メソッドを追加
- 3, 4, 5 以外の値をリジェクト

## 内部値とラベルの対応
| 内部値 | ラベル | 意味 |
|--------|--------|------|
| 3 | 観た | 観劇した記録 |
| 4 | 良かった | ポジティブ評価 |
| 5 | 最高 | 最高評価 |

## 注意事項
- migration は不要（DB上の型・制約は変更なし、アプリレベルのバリデーションのみ）
- 既存データで rating_overall が 1 or 2 のレコードがある場合、表示時にラベル変換されない（空文字になる）
- API レスポンスは従来通り数値で返す。ラベル変換はフロント側で行う
