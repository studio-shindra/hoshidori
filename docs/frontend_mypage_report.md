## 画面構成

### ヘッダー
- display_name + @username
- 「観た舞台や、これから観る舞台をまとめた自分の棚」

### サマリーカード（タブ兼用）
- これから（件数） — タップでタブ切替
- 観た（件数）
- 感想（件数）
- ポスター（件数）
- アクティブタブはボーダーでハイライト

### タブ: これから観る
- GET /api/viewing-logs/?status=planned
- 公演名、バッジ、メモ
- ポスターサムネ用プレースホルダー

### タブ: 観た
- GET /api/viewing-logs/?status=watched
- 公演名、バッジ、観劇日、メモ

### タブ: 感想
- GET /api/reviews/ → 自分の投稿のみフィルタ
- 公演名、星、本文（4行clamp）、いいね数、投稿日
- ネタバレバッジ対応

### タブ: ポスター
- 現状「自分のポスター一覧API」が未実装のため空状態表示
- UIの箱は完成済み（画像グリッド、採用中バッジ等）

### 空状態
- 各タブでデータ0件時に案内テキスト + 導線ボタン

### ログアウト
- ページ最下部

---

## 使用API

| API | 用途 |
|-----|------|
| GET /api/auth/me/ | ユーザー情報（auth store経由） |
| GET /api/viewing-logs/?status=planned | これから観る一覧 |
| GET /api/viewing-logs/?status=watched | 観た一覧 |
| GET /api/reviews/ | 全感想（自分のものをフィルタ） |
| POST /api/auth/logout/ | ログアウト |

---

## 確認手順

```bash
cd frontend && npm run dev
```

1. ログイン状態で `/mypage` にアクセス
2. サマリーカードに件数が表示される
3. 各カードをタップしてタブが切り替わる
4. データがある場合、一覧が表示される
5. データが0件の場合、空状態メッセージと導線が表示される
6. ログアウトボタンが動作する
