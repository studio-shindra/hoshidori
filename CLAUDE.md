# CLAUDE.md

## 基本方針
- 目的は、最小変更で素早く実装を進めること
- 不要な確認質問はしない
- まず既存コードを読んで、影響範囲を把握してから編集する
- 関係ないファイルは触らない
- 大規模リファクタは禁止
- TypeScriptは使わない
- Pythonは必ず `python3` を使う
- 日本語で簡潔に返答する

## Bash運用ルール
- Bash実行は本当に必要な場合だけにする
- 読み取りと編集を優先し、不要なコマンド実行はしない
- build, test, lint, git status などの定型確認は、原則として実行せず、必要ならコピペ可能なコマンドを1つにまとめて提示する
- Bashが必要な場合は、細かく何度も叩かず、必要最小限にまとめる
- 長時間コマンド、破壊的コマンド、副作用のあるコマンドは勝手に実行しない
- 実装後の確認が必要な場合は、まず「実行コマンド」を提示する

## 実装ルール
- 変更は最小限
- 既存設計を尊重する
- 未使用importは消す
- 関係ない整形だけの変更は禁止
- 勝手なライブラリ追加は禁止
- 勝手なmigrationは禁止
- 説明は短く、変更ファイル名を先に出す
- コードはコピペ可能な形で出す
